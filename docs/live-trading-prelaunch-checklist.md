# Earnings Gap-Up Strategy 本番前チェックリスト（1ページ）

対象戦略: `report_open` エントリー / `intraday` ストップ / 最大90日保有
最終更新: 2026-02-15

## 0. Go/No-Go 判定（必須ゲート）

- [ ] **データ健全性**: 同一入力でバックテスト結果が再現する（KPI差分なし）
- [ ] **テスト**: `pytest backtest/tests/` が全件パス
- [ ] **OOS検証**: 直近OOSで `PF >= 1.30`
- [ ] **コスト感応度**: スリッページ強化ケースでも `PF >= 1.10`
- [ ] **リスク**: 想定最大DDが運用許容内（例: 総資金の`<= 15%`）
- [ ] **実行可能性**: `Capital Required` が実運用資金枠内
- [ ] **障害対策**: 緊急停止手順・代替手順（手動クローズ）が文書化済み

> 1つでも未達なら **No-Go**。本番投入しない。

## 1. 戦略仕様凍結（変更管理）

- [ ] エントリー仕様を固定（`--entry-mode report_open`）
- [ ] ストップ仕様を固定（`--stop-mode intraday`、`--stop-loss 10`、`--slippage 0.5`）
- [ ] フィルタ仕様を固定（例: `--daily-entry-limit 2 --min-score 55 --max-score 85 --min-grade B`）
- [ ] 仕様変更時は「変更理由・期待効果・比較結果」を `docs/` に記録
- [ ] 本番実行コマンドを固定し、再現可能な形で保存

## 2. 実行前（T-1日〜当日寄り前）

- [ ] FMP APIキー有効 (`FMP_API_KEY`) / レート制限確認
- [ ] 当日レポート生成完了（対象HTMLの欠損なし）
- [ ] パーサ結果確認（候補件数、`no_score`、`skipped` が異常値でない）
- [ ] その日の新規シグナル件数が想定レンジ内
- [ ] 注文上限設定（1銘柄上限、同時保有上限、日次損失上限）
- [ ] 取引不可銘柄（流動性不足・売買停止）除外ルール適用

## 3. 当日執行チェック

- [ ] 発注前に現金余力・証拠金余力を確認
- [ ] 注文方法（成行/指値）と失効条件を確認
- [ ] 約定ログ（時刻・価格・数量）を保存
- [ ] 未約定・部分約定の処理ルールを適用
- [ ] 異常時は新規建て停止（API障害、価格配信異常、急変動）

## 4. 日次モニタリング（EOD）

- [ ] 実現/含み損益を記録（バックテスト想定との差分も記録）
- [ ] ストップ発動件数と理由を確認（想定外急増の検知）
- [ ] 主要KPI更新: Win Rate / PF / MaxDD / Capital Utilization
- [ ] バックテストとの乖離原因を分類（約定差、データ差、ルール逸脱）
- [ ] 乖離が閾値超過なら翌営業日を縮小運用または停止

## 5. Paper Trading 完了条件（本番前）

- [ ] 連続`20`営業日以上、同一ルールで運用ログ取得済み
- [ ] 重大障害（注文失敗放置、誤発注、データ欠損）が`0`件
- [ ] 期待値が維持（PF・DDが許容範囲）
- [ ] 運用手順に手戻りがない（手作業依存が解消）

## 6. 本番投入と段階拡大

- [ ] Day 1-10: 想定資金の`10-20%`で開始
- [ ] Day 11-20: KPIが基準内なら`30-50%`へ拡大
- [ ] Day 21+: KPI安定後にフルサイズへ
- [ ] 拡大条件未達なら直前ステップへ戻す
